Complete Community Platform Requirements & Design Specification
I. Project Goal & Philosophy

Core Mission

Goal: Create a supportive community platform for caregivers
Design Philosophy: Professional yet warm, mirroring the existing app's design to create a trustworthy and inviting environment
Target Audience: Women aged 45-65, who are primary caregivers
Platform Strategy: Mobile-first, but desktop-friendly
Monetization: Advertiser partnerships with healthcare/service companies

II. Existing App Design Elements (to be mirrored)

Color Palette System

Primary Colors: Focus on purple (e.g., purple-400/500) and orange (e.g., amber-400/500) for primary actions and visual emphasis
Category-Specific Gradients: Implement category-specific gradients similar to the existing app:
Senior Living: from-rose-400 to-pink-500 with bg-rose-50 background
Home Care: from-blue-400 to-blue-500 with bg-blue-50 background
Government: from-emerald-400 to-emerald-500 with bg-emerald-50 background
Attorneys: from-amber-400 to-orange-500 with bg-amber-50 background
Other Professionals: from-indigo-400 to-indigo-500 with bg-indigo-50 background
VA: from-teal-400 to-teal-500 with bg-teal-50 background
Physical Therapy: from-cyan-400 to-cyan-500 with bg-cyan-50 background
Paying for Care: from-lime-400 to-lime-500 with bg-lime-50 background
Status Colors: Use the existing status color system:
Purple: Unread items, primary actions
Amber/Orange: Pending replies, warnings
Red: Urgent items, no response yet
Green: Completed items, success states
UI Foundation Colors:
Backgrounds: White cards, gray-50 hover states
Text: Gray-800 for primary text, gray-500 for secondary
Borders: Gray-200 for card borders
Interactive: Hover shadows and subtle transforms

Visual Design Elements

Overall Aesthetic: Professional yet warm, sophisticated with soft edges and pleasant gradients
Typography: Clean, readable font (based on existing system font stack)
Button Styling: Rounded edges (following existing rounded-2xl and rounded-lg patterns)
Card Design:
Rounded corners (rounded-2xl for main cards)
Subtle shadows (shadow-sm with hover shadow-md)
White backgrounds with border (border-gray-200)
Hover effects with slight elevation (hover:translate-y-[-4px])
Circle Elements:
Status indicators as circles (w-8 h-8 to w-9 h-9)
Gradient backgrounds on circular elements
Small white dots inside circles for visual interest
Layout Patterns:
Grid layouts (responsive: grid-cols-1 sm:grid-cols-2 lg:grid-cols-3)
Card-based information architecture
Consistent spacing (gap-6 sm:gap-8 lg:gap-10)
Padding patterns (p-4 sm:p-6)

Security & Safety Protocol

Email/Contact Protection Rules:

No Email Addresses: Auto-detect and flag/remove email addresses in posts
Company Website Only: Experts can mention "Visit CompanyName.com" but no direct contact
Link Safety Warning: "This link leads outside our community. Please verify the source before sharing personal information."
Verification Badge: Only verified experts can share business websites
Content Moderation for Links:

interface LinkSafetyProtocol {
  allowedDomains: string[]; // Pre-approved healthcare, government sites
  requiresWarning: boolean; // All external links get safety notice
  expertOnly: boolean; // Only verified experts can share business links
  autoFlag: {
    emailAddresses: boolean;
    phoneNumbers: boolean;
    socialMediaHandles: boolean;
    directContactRequests: boolean;
  }
}
No Private Messaging Initially:

Only public posts allowed at launch
Prevents spam, scams, and inappropriate contact
Maintains transparency and community oversight
All support and advice happens in public forum format
